# 智学AI 开发约束

本文件约束智学AI的产品开发、AI调用、数据处理、安全和上线流程。当前版本以真实可用功能为准，不保留演示数据、占位功能或本地假AI回退。

## 1. 产品原则

- 不直接给最终答案。
- 不代写作业。
- 不诱导作弊。
- 优先保护未成年人隐私和安全。
- 功能必须真实持久化，刷新后数据不能丢。
- 页面上不能出现演示账号、演示题目、演示班级、演示作业。
- 空状态必须说明下一步真实操作，不能伪造数据。

## 2. AI 调用约束

- 前端不得直接接触 API Key。
- 所有 AI 请求必须通过服务端代理。
- 服务端必须从环境变量读取 `AI_API_KEY`、`AI_API_BASE` 和 `AI_MODEL`。
- 缺少 API Key 时必须明确报错，不允许返回本地假答案。
- 流式对话必须使用流式输出。
- JSON 类 AI 响应必须解析为结构化数据后再入库。
- AI 不确定知识点时返回空数组，后端可标记为 `待AI归类`，不得关键词硬猜。
- Markdown 和数学公式统一使用：
  - 行内公式：`$...$`
  - 独立公式：`$$...$$`

## 3. 安全约束

- `.env` 不得提交、不得打包、不得写入文档。
- 日志不得输出 API Key、JWT、密码或完整隐私数据。
- 密码必须哈希存储。
- JWT 密钥必须来自环境变量。
- 老师、家长、学生、管理员权限必须分离。
- 公开注册不得直接授予管理员身份。
- 家长和老师只能访问授权范围内的数据。
- 生产部署必须配置 `APP_ORIGINS` 白名单，不允许任意来源访问 API。
- 登录、注册和 AI 重接口必须有限流保护。
- 文件上传必须限制类型和大小。
- OCR 图片不得保存为公开可访问文件。
- 删除账号必须级联删除相关个人数据。

## 4. 数据约束

- 用户、会话、消息、错题、知识卡片、测验、计划、作业、报告、目标、公告必须持久化。
- 不允许用前端内存假装保存。
- 不允许用固定数组假装数据库。
- 知识地图由真实创建的数据驱动，不自动插入演示节点。
- 管理看板必须来自当前数据库聚合结果。

## 5. 前端约束

- 移动端优先。
- 输入栏不能被遮挡。
- 聊天内容必须可滚动。
- 发送、生成、上传、批改等耗时操作必须有加载反馈。
- 表单默认值为空或来自用户真实资料。
- 按钮在必填项为空时应禁用或给出明确反馈。
- 所有 AI 内容展示面支持 Markdown 和数学公式。
- 历史记录点击必须进入对应聊天。

## 6. 上线检查

上线前至少人工确认：

- 注册登录正常。
- 登录状态刷新后保留。
- AI 对话真实调用远程 API。
- 流式输出正常。
- 知识卡片刷新后保留。
- 学习计划刷新后保留。
- 测验生成和批改真实可用。
- 老师端班级、作业、公告真实持久化。
- 学生端可以查看并提交作业。
- 家长端只能查看授权学生报告。
- 知识地图空状态和新增流程正常。
- 移动端菜单、输入框、滚动区域正常。
- 压缩包不包含 `.env`、数据库、日志、依赖目录和构建缓存。

## 7. 禁止事项

- 禁止在正式代码里写演示账号。
- 禁止内置演示题目作为默认输入。
- 禁止本地 AI 假回退。
- 禁止关键词硬编码判断知识点。
- 禁止把测试数据混入生产数据库。
- 禁止把 API Key 写进前端、文档、日志或压缩包。
- 禁止移除或弱化专有商业许可证。
